Hypertext Transfer Protocol (HTTP) & Hypertext Transfer Protocol Secure (HTTPS)

The Digital "Please," "Thank You," and the "Bodyguard."

HTTP is the basic rulebook for how browsers and servers talk. When your browser wants a page, it sends an "HTTP Request." When the server sends it back, that's an "HTTP Response." If the server can't find it, it gives you the famous "404 Error"—which is just nerd-code for "I have no idea what you're talking about."

But what about the 'S'?

HTTPS is the "Secure" version. Think of it as HTTP wearing a tuxedo and carrying a briefcase. It uses Encryption (Secure Sockets Layer/Transport Layer Security (SSL/TLS)) to scramble your data so that if a hacker tries to peek at your password or credit card info, it looks like a bunch of random, unreadable symbols.
